# lite-interpreter 前端目标设计

这份文档是 `lite-interpreter` 前端的目标样式与交互约束说明。它不是“效果图复述”，而是把三张参考图收敛成一个**可实现、可迭代、无幻想数据**的产品前端标准。

## 设计目标

- 让前端看起来像一套真正的企业级分析控制台，而不是通用 AI 看板
- 同时服务两类使用者：业务分析人员和运行时/治理观察者
- 所有展示都必须来自真实 API 或明确的可推导数据，不伪造 DAG、节点结果或统计指标
- 用统一的高奢深色视觉语言，把“资料 → 证据 → 分析 → 产物 → 复核”串成一条可信链路

## 三个目标界面

### 1. 业务工作台

对应参考图 `lite-interpreter-v2-03-role-based-command-center` 和 `lite-interpreter-v2-01-cfo-workbench`。

这是面向财务、会计、经营分析用户的主入口，重点展示：

- 新建分析的快速入口
- 最近分析的状态、摘要、风险提示
- 资料、方法、审计等支持模块的入口
- 清晰但克制的右侧状态栏

### 2. 运行时透明度

对应参考图 `lite-interpreter-v2-02-runtime-observability`。

这是面向内部观察者、治理人员和工程人员的透明视图，重点展示：

- 真实的运行进度
- 事件流与状态变化
- 产物和预览
- 风险、警告、手工介入点

### 3. 角色化指挥中心

这是前两者的统一壳层：同一个产品，同时支持业务视图和运行时视图，但不同角色看到的重点不同。

## 视觉语言

- **基底**：深石墨 / 近黑背景，避免纯黑；卡片与面板使用层次化深灰
- **主强调**：香槟金 / 古铜金，用于主 CTA、标题强调、分割线与关键状态
- **成功态**：翡翠绿，只用于已验证、已完成、允许等正向状态
- **信息态**：冰蓝，用于透明度、可观测、说明性内容
- **警告态**：琥珀金，用于待确认、风险提示、人工介入
- **禁止态**：柔和红，只用于失败、拒绝、阻断

## 布局语法

### 全局壳层

- 左侧固定导航栏：品牌、工作区、主要模块、账号信息
- 顶部命令栏：面包屑、快速问题输入、会话状态、用户入口
- 中央内容区：页面主内容，使用卡片栈和双栏/三栏布局

### 页面分层

- **首屏**：一个强主标题 + 一个主行动入口 + 一个状态概览
- **第二层**：数据卡、流程卡、进度卡
- **第三层**：列表、表格、事件流、结果预览

### 卡片规则

- 主要容器使用大圆角、细边框、轻玻璃感
- 内部卡片比外层略亮一档，但不使用纯白
- 预览区可以更深，以强化“控制台”感

## 真实数据边界

### 当前前端可以直接展示

- `GET /api/app/session`
- `GET /api/app/analyses`
- `GET /api/app/analyses/{analysis_id}`
- `GET /api/app/analyses/{analysis_id}/events`
- `GET /api/app/analyses/{analysis_id}/outputs/{output_id}`
- `GET /api/app/assets`
- `GET /api/app/methods`
- `GET /api/app/audit`

### 只能作为内部视图或未来扩展，不可在业务页伪造

- 完整 DAG 可视化
- `Coder` / `Debugger` / `Executor` 之类节点级内部合同
- `NodeOutcome`、`TerminalVerdict`、`SandboxBoundary` 等工程内部细节
- 假造的产物数量、测试通过数、存储占用、覆盖率

## 页面映射

- `/analyses`：业务工作台首页，承接快速提问和最近分析
- `/analyses/new`：新建分析与资料选择
- `/analyses/:analysisId`：分析详情，支持业务视图 / 运行时视图切换
- `/assets`：资料库
- `/methods`：方法库
- `/audit`：治理与审计
- `/settings/session`：连接与会话

## 交互规则

- 顶部快速问题输入只能把用户带到真实的新建分析流程，不直接伪造执行结果
- 运行时视图只能使用当前分析真实返回的 `progress`、`events`、`outputs`、`warnings`
- 业务视图优先讲结论、证据、可复核产物
- 所有“高奢感”必须建立在秩序、克制和信息层级上，而不是靠大量装饰元素

## 结构化输入边界

当前稳定支持的结构化输入只应被视为：

- `csv`
- `tsv`
- `json`

`xlsx`、`xls`、`parquet` 等只可作为未来扩展或业务文档示例，不应在业务页宣称为稳定静态分析输入。

## 结论

这套 UI 的目标不是“复制参考图”，而是把参考图里的布局秩序、信息分层和高奢质感，转化成一个能长期扩展的企业级分析前端。
