# lite-interpreter 产品需求文档 (PRD)

> 文档版本：1.0 | 最后更新：2026-04-29 | 状态：初稿

---

## 1. 产品概述

### 1.1 产品定位

`lite-interpreter` 是一个面向**财务、会计与经营分析场景**的受控分析运行时平台。它不是一个自由发挥的通用自主 Agent，而是一个以"**可治理、可观测、可回放、可交付**"为目标的工程化数据分析系统。

### 1.2 核心价值主张

| 维度 | 描述 |
|------|------|
| **可治理** | DAG 静态编排 + Harness 治理策略，分析过程可控、可审计 |
| **可观测** | Blackboard 状态事实源 + Event Bus 实时事件流，全程透明 |
| **可回放** | Event Journal 记录完整执行历史，支持事后复盘与审计 |
| **可交付** | 结构化产物输出，结果可下载、可引用、可追溯 |

### 1.3 产品形态

- **Web 工作台**：React/Vite 单页应用，提供完整的分析操作界面
- **App-facing API**：`/api/app/*` RESTful 接口，支持程序化调用
- **核心运行时**：Python 后端，DAG 编排 + Sandbox 执行 + 知识增强

## 2. 目标用户

### 2.1 主要用户画像

| 用户角色 | 典型场景 | 核心需求 |
|----------|----------|----------|
| **财务分析师** | 利润分析、成本核算、财务报表解读 | 快速分析数据、生成结构化报告 |
| **会计人员** | 账务核查、科目对账、合规检查 | 可审计的分析过程、可追溯的结果 |
| **经营管理者** | 经营指标监控、趋势分析、决策支持 | 易于理解的结论、可靠的分析方法 |
| **数据工程师** | 分析流程自动化、方法沉淀与复用 | API 集成、方法库管理 |

### 2.2 用户能力假设

- 能够上传结构化数据文件（CSV/TSV/JSON）
- 能够用自然语言描述分析需求
- 不需要编程能力即可使用主产品面

## 3. 功能需求

### 3.1 P0 - 核心功能（主闭环必须）

#### 3.1.1 会话与工作区管理

- **F-001**：通过 Bearer Token 完成 API 认证
- **F-002**：通过 `/api/app/session` 获取会话信息与可用工作区
- **F-003**：支持多工作区切换，每个工作区独立的资料库与分析空间
- **F-004**：角色权限控制（viewer / operator / admin）

#### 3.1.2 资料管理

- **F-010**：支持上传 CSV、TSV、JSON 格式的结构化数据文件
- **F-011**：资料按工作区隔离，支持资产列表查看
- **F-012**：创建分析时通过 `assetIds` 显式挂接输入资料
- **F-013**：资料元数据管理（文件名、大小、格式、上传时间）

#### 3.1.3 分析创建与执行

- **F-020**：通过自然语言问题创建分析任务
- **F-021**：DAG 静态链自动编排执行（router → data_inspector → analyst → coder → auditor → executor → summarizer）
- **F-022**：执行过程实时状态更新（通过事件轮询）
- **F-023**：执行结果自动投影为可读的分析结论
- **F-024**：支持 `analysisModePreset` 指定分析模式

#### 3.1.4 结果消费

- **F-030**：分析列表查看（分页、排序）
- **F-031**：分析详情查看（结论、状态、时间线）
- **F-032**：分析事件轮询（实时进度追踪）
- **F-033**：结果产物查看与下载
- **F-034**：分析摘要自动生成

#### 3.1.5 审计与回放

- **F-040**：审计记录列表（分页查询）
- **F-041**：完整执行过程可回放
- **F-042**：操作日志与变更追踪

### 3.2 P1 - 增强功能（显著提升体验）

#### 3.2.1 动态研究能力

- **F-100**：DeerFlow sidecar 动态研究编排
- **F-101**：多步探索式研究（需要迭代搜索、外部资料检索的复杂任务）
- **F-102**：动态研究轨迹标准化与回写
- **F-103**：动态研究结果回流静态链继续处理

#### 3.2.2 知识增强（KAG）

- **F-110**：业务文档解析与知识图谱构建
- **F-111**：混合检索（图检索 + BM25 + SPLADE 向量检索）
- **F-112**：上下文压缩与选择，为 LLM 提供高质量上下文
- **F-113**：证据召回与去重

#### 3.2.3 方法沉淀（SkillNet）

- **F-120**：成功分析路径自动沉淀为可复用方法
- **F-121**：方法校验与质量评估
- **F-122**：方法提升（从实验级到推荐级）
- **F-123**：历史方法检索与复用

#### 3.2.4 工作区资料管理

- **F-130**：资料列表与详情查看
- **F-131**：资料与分析的关联关系追踪

### 3.3 P2 - 扩展功能（未来演进）

- **F-200**：更多数据格式支持（xlsx、parquet — 需完整闭环验证）
- **F-201**：丰富的产物预览（图表、可视化）
- **F-202**：长任务细粒度进度反馈
- **F-203**：协作功能（分享分析、团队工作区）
- **F-204**：分析模板与批量执行

## 4. 非功能需求

### 4.1 安全性

| 需求 | 描述 |
|------|------|
| **多租户隔离** | 数据按 tenant/workspace 隔离，路径强制校验 |
| **代码审计** | AST 静态分析阻止危险代码执行 |
| **沙箱隔离** | Docker 容器内执行用户代码，限制资源访问 |
| **认证授权** | Bearer Token 认证 + 角色权限模型 |
| **产物路径安全** | 结果下载限制在当前任务的 tenant/workspace 目录内 |

### 4.2 可观测性

| 需求 | 描述 |
|------|------|
| **实时事件流** | Event Bus 提供分析执行的实时事件推送 |
| **历史回放** | Event Journal 支持完整执行过程的事后回放 |
| **状态透明** | Blackboard 作为唯一事实源，所有模块读写一致 |
| **审计日志** | 关键操作自动记录到审计系统 |

### 4.3 性能

| 需求 | 描述 |
|------|------|
| **响应时间** | API 列表查询 < 500ms，详情查询 < 1s |
| **并发支持** | 支持多任务并行执行（DAG 编排） |
| **资源控制** | Sandbox 执行限制 CPU/内存/时间 |

### 4.4 可靠性

| 需求 | 描述 |
|------|------|
| **错误处理** | 统一结构化错误 envelope，明确错误码与消息 |
| **任务恢复** | 支持任务中断后的检查点恢复 |
| **降级策略** | 关键服务不可用时明确返回 `unavailable`，不静默回退 |

### 4.5 可扩展性

| 需求 | 描述 |
|------|------|
| **节点扩展** | DAG 节点可独立新增和替换 |
| **存储扩展** | Repository 层抽象，支持替换存储后端 |
| **模型扩展** | LLM 客户端支持多模型配置（LiteLLM） |

## 5. 产品边界与非目标

### 5.1 明确不是什么

- ❌ 不是通用自主 Agent（不自由发挥，受 DAG 编排约束）
- ❌ 不是"一个 prompt 解决所有问题"的黑箱助手
- ❌ 不是 DeerFlow 的 runtime playground（DeerFlow 只是 sidecar）
- ❌ 不是支持多种格式但闭环不了的"假支持"产品

### 5.2 当前不做的事

- 不恢复已废弃的 Streamlit 前端
- 不恢复旧 `tasks/executions/uploads` 风格公开接口
- 不把 embedded/auto runtime mode 作为正式能力
- 不继续宣传未完成闭环的格式支持（xlsx/xls/parquet）
- 不把辅助页面包装成比主分析链路更重要的"版本卖点"

## 6. 用户旅程

### 6.1 核心旅程：从数据到分析结论

```
用户打开 Web 工作台
    │
    ▼
登录（Bearer Token 配置）
    │
    ▼
选择 / 创建工作区
    │
    ▼
上传资料（CSV/TSV/JSON）
    │
    ▼
创建分析任务（输入问题 + 选择资料）
    │
    ▼
系统自动编排执行
    ├─ 静态链：数据检查 → 知识检索 → 分析 → 编码 → 审计 → 执行
    └─ 动态链：DeerFlow 研究 → 回流静态链
    │
    ▼
实时查看执行进度（事件轮询）
    │
    ▼
查看分析结论与结果产物
    │
    ▼
下载产物 / 查看审计记录
    │
    ▼
成功路径自动沉淀为可复用方法
```

### 6.2 辅助旅程

- **方法复用旅程**：查看历史方法 → 选择适用方法 → 应用到新分析
- **审计回放旅程**：选择已完成分析 → 查看执行事件时间线 → 回放关键步骤
- **资料管理旅程**：查看工作区资料 → 上传新资料 → 关联到分析任务

## 7. API 合同概览

### 7.1 核心端点

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/app/session` | 会话 bootstrap |
| GET | `/api/app/analyses` | 分析列表（分页） |
| POST | `/api/app/analyses` | 创建分析 |
| GET | `/api/app/analyses/{id}` | 分析详情 |
| GET | `/api/app/analyses/{id}/events` | 分析事件流 |
| GET | `/api/app/analyses/{id}/outputs/{oid}` | 结果产物内容 |
| GET | `/api/app/assets` | 资料列表 |
| POST | `/api/app/assets` | 上传资料 |
| GET | `/api/app/methods` | 方法列表 |
| GET | `/api/app/audit` | 审计记录 |

### 7.2 合同约定

- 所有列表接口支持 `page` / `pageSize` 分页
- 错误统一返回结构化 `{ code, message, details }` envelope
- 认证通过 `Authorization: Bearer <token>` 头
- 工作区通过 `workspaceId` query 参数切换

## 10. 术语表

| 术语 | 定义 |
|------|------|
| **DAG** | Directed Acyclic Graph，有向无环图，用于分析任务编排 |
| **Blackboard** | 黑板架构，系统状态的唯一事实源 |
| **KAG** | Knowledge-Augmented Generation，知识增强生成 |
| **SkillNet** | 方法沉淀与复用网络 |
| **DeerFlow** | 动态研究 sidecar，负责复杂探索式研究 |
| **Sandbox** | Docker 沙箱，安全执行用户代码 |
| **Harness** | 治理层，控制动态执行的策略与边界 |
| **静态链** | DAG 中确定性编排的分析路径 |
| **动态链** | DAG 中需要迭代探索的分析路径 |
