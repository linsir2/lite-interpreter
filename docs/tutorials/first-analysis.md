# lite-interpreter 业务使用手册

这是一份面向 **财务、会计、经营分析、业务复核人员** 的使用说明。

它不是开发文档，不讲架构、部署和测试；它只回答一件事：

> 我拿到这个平台之后，怎样从“上传资料”走到“拿到可复核的分析结果”。

当前产品状态、能力边界与已知限制统一以 `docs/reference/project-status.md` 为准；本文件只讲使用流程。

## 0. 五分钟上手

如果你只想最快跑通一次分析，按这个顺序做：

1. 找管理员拿到 **API 地址** 和 **Bearer Token**
2. 打开前端，先完成“连接真实分析前端”
3. 确认左侧栏当前工作区是对的
4. 去 **资料库** 上传 2 到 4 份本次分析真正需要的资料
5. 回到 **分析总览**，点 **新建分析**
6. 写清楚问题，并手动勾选本次要引用的资料
7. 提交后进入详情页，看 **结论摘要**、**关键发现**、**进度事件** 和 **结果产物**

如果这是你第一次正式用平台，强烈建议从一个范围明确、资料干净的小问题开始。

## 1. 你开始前需要准备什么

进入平台前，通常需要向系统维护者拿到两样东西：

1. **API 地址**
   - 例如：`http://127.0.0.1:8000`
2. **Bearer Token**
   - 这是你的访问凭证
   - 不同 token 对应不同工作区和权限

如果你看不到某些页面，通常不是系统坏了，而是你的 token 没有对应权限。

## 2. 第一次进入平台

打开 Web 前端后，先会看到“连接真实分析前端”页面。

你需要填写：

- **API 地址**
- **Bearer Token**

然后点击 **“进入分析平台”**。

进入后，系统会自动读取你的会话信息，并根据权限显示可访问的工作区。

## 3. 工作区是什么意思

工作区可以理解为一组相对独立的分析空间。

每个工作区通常有自己的一组：

- 资料
- 分析任务
- 方法沉淀
- 审计记录

如果你切换了工作区，页面中的列表也会一起切换。因此开始分析前，先确认左侧栏显示的是正确工作区。

## 3.1 页面导览

左侧主导航通常会看到这些页面：

- **分析总览**
  - 看最近分析、待跟进任务，并进入新建分析
- **资料库**
  - 上传和查看工作区资料
- **分析方法**
  - 查看当前工作区已经沉淀的分析方法
- **审计记录**
  - 查看关键操作和结果，通常需要更高权限
- **连接与会话**
  - 修改 API 地址、Bearer Token，重新连接平台

如果你只是正常做分析，最常使用的其实只有三个页面：

1. **资料库**
2. **分析总览**
3. **分析详情**

## 4. 一次完整分析怎么做

最推荐的顺序是：

1. 先上传本次分析需要的资料
2. 再新建分析任务
3. 在新建任务时只勾选本次真正要引用的资料
4. 等分析完成后查看结论、事件和结果产物

不要反过来操作，也不要把整个工作区所有资料都一股脑挂上去。

## 5. 第一步：上传资料

进入左侧导航的 **“资料库”** 页面。

你会看到两个区域：

- 左边：上传资料
- 右边：当前工作区资料列表

页面里常见的判断方式是：

- 左边负责“把资料放进工作区”
- 右边负责“确认这份资料是否已经准备好供分析使用”

### 5.1 选择资料类型

上传前先选资料类型：

- **结构化数据**
  - 适合报表、台账、明细表
- **说明文档**
  - 适合制度、业务说明、口径文档、分析背景材料
- **自动识别**
  - 让系统自己判断

### 5.2 当前最稳妥的结构化格式

如果你上传的是分析数据，当前最稳妥的是：

- `csv`
- `tsv`
- `json`

如果你手里是 Excel，建议先让维护者或数据同学预转换成 `csv` 再分析。这样成功率和可复核性更高。

### 5.3 上传时的建议

- 一次只上传与你当前分析目标真正相关的资料
- 文件命名尽量带时间范围和主题
  - 例如：`2026-03-profit-detail.csv`
- 说明文档尽量写清业务口径和字段含义

### 5.4 看到“可直接分析”是什么意思

资料列表里如果显示类似 **“可直接分析”** 的状态，说明这份资料已经达到当前链路可直接使用的准备程度。

如果不是这个状态，通常意味着：

- 文件类型不理想
- 数据还需要清洗
- 资料暂时只适合留在工作区，不适合直接进入本次分析

## 6. 第二步：新建分析

进入 **“分析总览”** 页面，点击右上角 **“新建分析”**。

新建页分成两个核心部分：

1. 左边：问题和分析策略
2. 右边：本次要引用的资料

你可以把这个页面理解为：

- 左边决定“你想问什么”
- 右边决定“系统可以基于什么回答”

### 6.1 怎么写问题

问题越具体，结果越容易复核。

推荐写法：

- 明确时间范围
- 明确分析对象
- 明确你要的输出形式

好的例子：

- “请对比 2026 年 3 月与 2 月利润表，解释利润下降的主要原因，并列出需要优先核对的异常项目。”
- “请结合费用明细和销售数据，说明本月毛利率下降是否主要来自折扣增加、销量变化还是成本上升。”

不好的例子：

- “帮我分析一下公司情况”
- “看看哪里有问题”

### 6.2 分析策略怎么选

页面里会提供若干分析策略预设，例如：

- `Researcher`
- `Reviewer`
- `Planner`

如果你没有明确偏好，可以先用默认策略。

一般理解为：

- **Researcher**：更偏探索原因和线索
- **Reviewer**：更偏核验、复核和风险控制
- **Planner**：更偏结构化拆解

### 6.3 为什么要手动勾选资料

这是平台很重要的一条设计：

> 上传到工作区，不等于自动进入每一次分析。

新建分析时，你必须手动勾选本次真正要引用的资料。

这样做的目的，是避免无关资料污染结论。

如果你发现结果跑偏，第一件要检查的事，往往不是模型，而是：

- 你有没有漏选关键资料
- 你有没有选进不相关资料

## 6.4 一个推荐示例

如果你是财务或经营分析同学，第一次可以直接照这个模版试：

- 问题：
  - “请对比本月与上月利润表，解释利润下降的主要原因，并列出需要优先核对的异常项目。”
- 资料：
  - 本月利润表
  - 上月利润表
  - 费用明细
  - 销售明细或收入说明

先把这个基础问题跑通，再增加更复杂的资料和目标。

## 7. 第三步：查看分析过程

分析创建后，会出现在 **“分析总览”** 里。

你可以从这里看见：

- 当前工作区总任务数
- 最近分析
- 需要继续跟进的任务

点进某个分析后，会进入详情页。

详情页最重要的区域有四块：

1. **结论摘要**
2. **关键发现**
3. **证据索引**
4. **进度事件**

如果页面右侧有结果产物，也要一起看，因为很多真正要留档或二次核对的内容会在那里。

## 8. 怎样读分析详情

### 8.1 先看结论摘要

结论摘要回答的是：

- 这次分析最后得出了什么结论
- 当前最推荐的下一步是什么

如果你只想快速判断“值不值得继续看”，先看这里。

### 8.2 再看关键发现

关键发现适合回答：

- 到底发现了哪几条异常
- 哪些点需要复核
- 结论背后有哪些主要支撑点

### 8.3 再看证据索引

证据索引的意义不是“装饰”，而是帮助你判断：

- 结论是否有依据
- 是否可以继续追溯到资料和执行结果

如果一个结论看起来很重，但证据索引很弱，你就应该更谨慎。

### 8.4 最后看进度事件

进度事件适合排查：

- 任务卡在哪一步
- 是否发生失败
- 最近系统正在做什么

如果任务一直没有结果，不要反复新建任务，先看事件。

## 9. 结果产物怎么理解

详情页右侧会显示 **“结果产物”**。

这些产物通常代表：

- 分析输出文件
- 中间结果
- 可下载的结果载体

如果页面提供“下载”按钮，说明当前产物可以通过受控接口读取或下载。

正确的使用方式是：

- 先看摘要判断有没有价值
- 再下载真正需要留档或二次复核的产物

如果你只想知道“结论是否可信”，不用第一时间下载所有产物；先看摘要、关键发现和证据索引更有效。

## 10. 方法库和审计页是做什么的

### 10.1 分析方法库

**“分析方法”** 页面主要给你看：

- 当前工作区已经沉淀了哪些可复用方法
- 这些方法用得多不多
- 它们需要哪些能力

它更像“经验资产列表”，不是你每次都必须操作的地方。

### 10.2 审计记录

**“审计记录”** 页面主要给管理者、管理员或需要复核的人使用。

它用来查看：

- 做了什么动作
- 是谁做的
- 结果成功还是失败
- 发生在什么时候

如果你看不到这个页面，通常是权限原因。

## 11. 常见问题

### 11.1 为什么我已经上传了资料，新建分析时还是没用上？

因为上传到工作区只是把资料放进资料库。

真正创建分析时，还需要在新建页面里手动勾选这些资料。

### 11.2 为什么我看不到审计页？

通常是因为当前 token 没有对应权限。

### 11.3 为什么结果看起来不对？

先按这个顺序排查：

1. 问题是否写得太模糊
2. 本次勾选的资料是否正确
3. 是否缺少关键说明文档或业务口径
4. 详情页事件里是否有失败或中断

### 11.4 为什么 Excel 文件不好用？

当前静态分析链最稳妥的是 `csv/tsv/json`。Excel 不是完全不能处理，但不应当被当作最稳定输入格式。

### 11.5 我应该一次上传多少资料？

只上传本次分析真正需要的那部分。

资料越杂，结论越容易被无关信息干扰。

### 11.6 我连不上平台怎么办？

优先检查：

1. API 地址是否正确
2. Bearer Token 是否复制完整
3. 你的 token 是否已经失效
4. 当前 API 是否真的已经启动

## 12. 推荐工作方式

如果你是第一次正式使用，建议按下面节奏来：

1. 先在小范围工作区做一次主题明确的分析
2. 只用 2 到 4 份最关键资料
3. 先验证平台是否能给出“可复核”的结论
4. 再逐渐增加更多资料和更复杂问题

这个平台的价值，不在于一次性吞下所有东西，而在于：

> 让一次分析任务从资料、过程到结果，都尽量可解释、可追踪、可复核。
